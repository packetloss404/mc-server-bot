# Delivery Roadmap

## Phase 0 - Preparation

- align on shared control vocabulary
- freeze page ownership and UI surfaces
- confirm persistence strategy under `data/`

## Phase 1 - MVP Control Platform

- shared command model
- command lifecycle events
- upgraded quick actions
- per-bot command history

## Phase 2 - Visible Work Planning

- mission queue
- reprioritization and retry
- interrupt vs append
- bot detail diagnostics

## Phase 3 - Spatial Operations

- markers
- zones
- routes
- click-to-command

## Phase 4 - Fleet Operations

- multi-select
- squads
- batch commands
- mission assignment to multiple bots

## Phase 5 - Persistent Automation

- roles
- policies
- autonomy settings
- loadouts and allowed zones

## Phase 6 - High-Level Control

- commander console
- typed-plan preview
- command templates and routines

## Phase 7 - Production Hardening

- tests
- telemetry
- release cleanup
- performance review

## Suggested First Release Boundary

Ship a first major revamp after Phase 3 if needed. At that point the dashboard will already support:

- strong per-bot control
- visible mission queueing
- map-driven operations

That gives users a meaningful upgrade before fleet, role, and commander features are complete.
